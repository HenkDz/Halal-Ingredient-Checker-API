FAQ

Frequently Asked Questions

Everything you need to know about the Halal Ingredient Checker API.

What is the Halal Ingredient Checker API?

The Halal Ingredient Checker API is a REST API that checks whether food ingredients and products are halal, haram, or doubtful. It supports ingredient lookup by name or E-number, barcode scanning via Open Food Facts, and batch verification of up to 50 products. The ingredient database is sourced from major halal certification bodies including IFANCA, JAKIM, and MUI.

Is the Halal Ingredient Checker API free?

Yes! The API offers a free tier with 100 requests/day, 10 requests/minute, single ingredient checking, and barcode scanning. For higher usage, the Pro tier ($9/month) includes 10,000 daily requests, batch checking up to 50 items, and detailed results with scholarly sources. Enterprise plans are available for food manufacturers and platforms.

How do I check if an ingredient is halal?

Send a POST request to /api/v1/check with a JSON body containing a list of ingredient names or E-numbers. Each ingredient returns a halal, haram, or doubtful verdict along with a detailed explanation and scholarly source references. You can also look up individual ingredients via GET /api/v1/ingredient/{name} or scan product barcodes via GET /api/v1/barcode/{ean}.

What E-numbers are haram?

Common haram E-numbers include E120 (Carmine/Cochineal — derived from crushed insects), E441 (Gelatin — usually from pork), E542 (Edible Bone Phosphate), and some E471 variants depending on their source. The API covers all major food additives, colorants, emulsifiers, and preservatives with verdicts sourced from IFANCA, JAKIM, MUI, and other halal certification bodies.

Can I scan product barcodes for halal verification?

Yes! The API supports barcode scanning for EAN/UPC codes via GET /api/v1/barcode/{barcode}. It fetches product data from Open Food Facts, extracts all ingredients, and checks each one against the halal ingredient database. Pro and Enterprise users can also batch-check up to 50 barcodes in a single request via POST /api/v1/barcode/batch.
